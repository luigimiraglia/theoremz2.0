## CONTENT BLOCK TYPES (intermediate format ‚Äî for sections 3 and 4 only)

The `content` array contains blocks. Each block has a `type` field (not `_type` ‚Äî the post-processor handles that). The post-processor automatically inserts spacers between non-section blocks and an HR before section blocks. You do not add spacers or HR before sections.

### `text` block ‚Äî body paragraph

```json
{
  "type": "text",
  "spans": [
    { "text": "La ", "marks": [] },
    { "text": "derivata", "marks": ["blueBold"] },
    { "text": " misura la velocit√† di cambiamento di una funzione in un punto.", "marks": [] }
  ]
}
```

`spans` is a flat array of span objects. Each span has:
- `text`: the visible string for this span
- `marks`: array of zero or more mark names (see marks reference below)

**Empty text block** (used as visual breath between dense passages):
```json
{ "type": "text", "spans": [{ "text": "", "marks": [] }] }
```

## MARKS REFERENCE

Marks go in a span's `marks` array. They are plain strings ‚Äî no key management needed.

### Decorator marks

**Hierarchy of emphasis ‚Äî use each mark exactly as described, never mix them up:**

| Mark | What it highlights | Frequency |
|---|---|---|
| `blueBold` | A single important technical or scientific **word** (e.g., "derivata", "coefficiente angolare", "limite") | Sparingly ‚Äî max 2‚Äì3 per section. Only for the first mention of a key term. |
| `highlightBlue` | A complete **sentence** that is a key concept the student must remember (renders as yellow highlight) | Once or twice per section ‚Äî only the single most important statement |
| `bold` | A complete **sentence** that is even more critical than a highlighted sentence ‚Äî the most essential rule or definition | Once per section at most ‚Äî the absolute core idea |
| `italic` | Light emphasis: law names, parenthetical remarks, foreign terms | As needed |
| `redBold` | Explicit warning: a common mistake or something NOT to do | 1‚Äì2 per section |
| `exUnderline` | Labels the opening line of a worked example | Once per example |

**Usage rules:**
- `blueBold` applies to a single WORD or short phrase, never a full sentence.
- `highlightBlue` and `bold` apply to a COMPLETE sentence, never a single word.
- Do NOT highlight every interesting sentence ‚Äî only the 1‚Äì2 most critical per section.
- Never apply `highlightBlue` and `bold` to the same span.

Marks can be combined freely: `["bold", "italic"]`, `["blueBold", "inlineLatex"]`.

### `inlineLatex` ‚Äî math symbol in running text

Add `"inlineLatex"` to a span's `marks` array. The span's `text` IS the LaTeX code ‚Äî **pure LaTeX, no delimiters**.

```json
{ "text": "\\Delta x", "marks": ["inlineLatex"] }
```

**Use `inlineLatex` for every math symbol in body text**, including:
- single variables: `x`, `y`, `F`, `m`, `a`, `v`
- Greek letters: `\\alpha`, `\\Delta`, `\\pi`
- expressions: `\\Delta x`, `x^2`, `\\frac{1}{2}mv^2`
- numbers used as math quantities: `9.81`, `10^{-12}`
- units in formulas: `\\text{m/s}^2`

Can combine with decorators: `["inlineLatex", "bold"]`, `["inlineLatex", "italic"]`.

### `mathBlueBox` ‚Äî important formula highlighted in a blue box

Write the full formula as `$$...$$` inside the span's `text`. Apply `mathBlueBox` as a mark.

```json
{
  "type": "text",
  "spans": [{ "text": "$$F = ma$$", "marks": ["mathBlueBox"] }]
}
```

Use **at most twice** per section ‚Äî only for the single most important formula.

### Plain display formula in a text block (no box)

Write `$$...$$` as span text with no marks.

```json
{ "text": "$$(a+b)^2 = a^2 + 2ab + b^2$$", "marks": [] }
```

## LATEX RULES

### ‚úÖ CORRECT

```
displayFormula latex field ‚Üí no delimiters:
  "latex": "F = ma"
  "latex": "\\frac{1}{2}mv^2"

inlineLatex span text ‚Üí pure LaTeX code, NO delimiters:
  "text": "x"             marks: ["inlineLatex"]
  "text": "\\Delta t"     marks: ["inlineLatex"]
  "text": "f(x) = x^2"   marks: ["inlineLatex"]

  "text": "$$F = ma$$"          marks: ["mathBlueBox"]
  "text": "$$(a+b)^2 = ...$$"   marks: []
```

### ‚ùå WRONG ‚Äî do not do these

```

"latex": "$$F = ma$$"

// ‚ùå delimiters inside inlineLatex span text
{ "text": "\\(f(x) = x^2\\)", "marks": ["inlineLatex"] }
{ "text": "\\displaystyle { f(x) = x^2 }", "marks": ["inlineLatex"] }
{ "text": "$f(x) = x^2$", "marks": ["inlineLatex"] }

// ‚ùå raw math in plain text spans (no inlineLatex mark)
{ "text": "dove F √® la forza e m la massa", "marks": [] }   // F, m should each be inlineLatex spans

// ‚ùå $$...$$ inside inlineLatex span
{ "text": "$$x^2$$", "marks": ["inlineLatex"] }
```
